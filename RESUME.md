### WSA MES/ERP 기능 개발

2024.03 ~ 2024.12
역할: 백엔드 개발

#### 프로젝트 개요
기존에 회사에서 있던 프로젝트 관리 서비스가 있었으나 너무 기본적으로 CRUD만 관리하게끔 되어 있었기에 스마트팩토리 구현을 위해 
프로젝트 수주부터 설계, 생산, 출하까지 이어지는 전반적인 업무 흐름을 관리하는 ERP 및 MES 시스템을 설계 및 개발했습니다.
이후 운영 단계에서 신규 기능을 지속적으로 개발하는 동시에, 레거시 구조를 점진적으로 개선하여 시스템의 안정성과 확장성을 높였습니다.

#### 담당 업무

- 영업 , 생산 , 재고 , 구매 각 영업 별 모듈의 도메인 설계 및 기술 개발
- 생산 관리 ERP 시스템의 프로젝트, 아이템, 공정 생산 흐름 설계
- Command/Query 분리 구조 기반 API 구현
- Querydsl 기반 동적 조회 및 DTO 중심 API 구조 개선
- 재고 처리 로직의 안정성 개선

#### 성과

- 생산 흐름에 맞는 도메인 구조를 설계하고 API를 분리해 유지보수성과 확장성을 높였습니다.
- Querydsl 기반 동적 조회 구조를 도입해 복잡한 조회 요구사항에 유연하게 대응했습니다.
- 재고 처리 로직을 `afterCommit` 이벤트 방식으로 분리해 동시성 안정성을 확보했습니다.

### WSA MSA 인프라 전환

2025.02 ~ 2025.09
역할: 아키텍처 및 공통 인프라 개발

#### 프로젝트 개요

기존 자사 MES/ERP 서비스를 MSA 구조로 전환하는 과정에서 Gateway, Auth, EMS, LMS 및 공통 인프라 구축을 담당했습니다.
서비스 분리 자체보다, 서비스 간 데이터 정합성과 공통 진입 정책, 운영 자동화 체계를 만드는 데 집중했습니다.

#### 담당 업무

- 서비스 간 데이터 동기화 구조 설계
- Gateway 전역 인증 필터 및 동적 라우팅 구성
- 배포 자동화 및 개발 환경 표준화

#### 성과

- Outbox + Kafka Connect CDC 기반 데이터 동기화 구조를 구축해 서비스 간 연동 신뢰성을 높였습니다.
- Gateway 전역 인증 필터와 동적 라우팅을 통해 공통 진입 정책을 표준화했습니다.
- GitHub Actions와 Docker Compose 기반 배포 자동화를 구성해 신규 서비스 온보딩 비용을 줄였습니다.
